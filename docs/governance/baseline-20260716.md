## Known failures

1. `tests/test_peft_adapters.py::TestPeftModelDeviceAndDtype::test_molora_model_dtype`
   fails because `MoLoRAExpert.forward()` casts the input to float32 while
   `.half()` has already converted adapter weights to float16.
2. `ultralytics/cfg/models/26/yolo26-master-n.yaml` cannot be constructed:
   its SPPF arguments do not match the current `SPPF(c1, c2, k=5)` signature.
3. `ultralytics/cfg/default.yaml` contains duplicate keys and does not expose
   all MoLoRA dataclass fields through the default YAML surface.
