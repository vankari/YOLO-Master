## Architecture Overview
The core execution flow of YOLO-Master revolves around "model—training—validation—inference—export," with the data pipeline running throughout, and tracking and solutions layered as upper-level capabilities.

```mermaid
sequenceDiagram
participant User as "User"
participant Model as "Model Entry<br/>ultralytics/engine/model.py"
participant Trainer as "Trainer<br/>ultralytics/engine/trainer.py"
participant Validator as "Validator<br/>ultralytics/engine/validator.py"
participant Predictor as "Predictor<br/>ultralytics/engine/predictor.py"
participant Data as "Data Pipeline<br/>ultralytics/data/build.py"
participant Exporter as "Exporter<br/>ultralytics/engine/exporter.py"
User->>Model : Initialize model and config
Model->>Trainer : Start training flow
Trainer->>Data : Build dataset and augmentation
Trainer-->>Model : Training logs and metrics
Model->>Validator : Validation evaluation
Validator-->>Model : Validation results
Model->>Predictor : Inference prediction
Predictor-->>User : Detection results/masks/keypoints
Model->>Exporter : Export to multi-backend formats
Exporter-->>User : Export artifacts and compatibility report
```
